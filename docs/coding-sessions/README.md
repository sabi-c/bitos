# Coding Session Notes Guide

Use this folder to keep an auditable narrative of implementation sessions.

## Required for each substantial iteration

- Create a new file: `YYYY-MM-DD_session-XXX_short-title.md`
- Include:
  - objective
  - starting state
  - changes made
  - tests/checks run
  - open issues/blockers
  - next tasks (with Task IDs from `docs/planning/TASK_TRACKER.md`)

## Coordination with task tracker

After completing a session note:
- update `docs/planning/TASK_TRACKER.md` statuses
- append a row to the tracker's **Iteration Log**
- ensure commit hash and session note agree
